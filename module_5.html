SSD_PCOM7E

The Secure SoftwareDevelopment module provided an opportunity to translate requirements from a design brief into working software implmenting user authentication, file transfer and authorization to access files. The solution was also required to be highly resistant to cyber attack with the requirement to demonstrate both feature implementation and protection mechanims.
The project created a completely database driven solution that supported distributed services and implemented the commonly abused user management features outside the HTTP application non-priviledged user interface to reduce the attack surface. /p>

Module learning outcomes

Learning outcomes stated at the outset of the module, (listed below), were not specifically traceable to the assignment criteria as a one for one match, rather the intention was to interprete assignments and provide a point of view, which may require the e-portfolio reader to be somewhat familiar with computer science and cyber security matters.

  • Identify security risks that exist within a proposed software solution
  • Implement suitable mitigations for identified risks
  • Develop a software program to meet requirements outlined in a design brief
  • Participate in a simulated development team environment
